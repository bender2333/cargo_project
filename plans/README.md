# Plans

每轮改动计划单独一个文件，命名 `YYYY-MM-DD-<简短主题>.md`（如 `2026-06-08-stack-capacity.md`）。

- 一轮一个文件，**不往旧文件追加**。
- 由架构师（Claude）在与用户拍板后写入，用户转发给 Codex 执行。
- 讨论稿不放这里——讨论记录进 `decision.md`，拍板后才把定稿计划写成本目录下的新文件。
- 旧的 `review.md` 仅供历史查阅，不再新增内容。

每个计划文件应包含（不必含代码细节）：
- 每个子任务的：根因（file:line）、意图与边界、模块划分、验证标准 / 测试用例（可断言、编码业务意图）、风险与回归门槛。
- 执行顺序、提交粒度、必须跑的验证命令。
